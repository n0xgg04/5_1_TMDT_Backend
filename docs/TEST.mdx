Mục tiêu:
Xây dựng bộ test case cho chương trình/ứng dụng mà nhóm đã phát triển.

1. Nội dung yêu cầu
   Các nhóm cần thực hiện các nội dung sau:
   • Xác định rõ chức năng cốt lõi cần kiểm thử.
   • Xây dựng tối thiểu bộ test case bao gồm cả:
   o Test case hợp lệ (positive case).
   o Test case không hợp lệ (negative case).
   • Mỗi test case phải thể hiện:
   o Điều kiện đầu vào.
   o Các bước thực hiện.
   o Kết quả mong đợi.
   o Kết quả thực tế (nếu đã test).
   o Trạng thái (Pass/Fail).
2. Phạm vi test
   • Các chức năng cốt lõi của chương trình
   • Thực hiện test thủ công (hoặc bằng công cụ nếu có thể).
3. Yêu cầu nộp
   • Nộp file Excel với định dạng theo mẫu gửi kèm (Chưa cần điền thông tin các cột: Kết quả thực tế; Trạng thái (Pass/Fail) ; Ghi chú)
