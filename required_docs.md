| STT | Tài liệu                   | Nội dung nên có                                                                                                                                                                                                                                                                                                                    | Tester dùng để làm gì                                                                                  |
| --- | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| 1   | SRS / Requirement          | - Mục tiêu hệ thống<br>- Danh sách chức năng<br>- Quy tắc nghiệp vụ<br>- Phân quyền<br>- Điều kiện validate<br>- Flow xử lý<br><br>Ví dụ:<br>- Người dùng được đăng ký bằng email<br>- Password tối thiểu 8 ký tự<br>- Giỏ hàng chỉ checkout khi còn hàng<br>- Admin mới được sửa sản phẩm<br>- Voucher chỉ áp dụng khi đơn > 500k | Dựa vào đây để tạo:<br>- Positive testcase<br>- Negative testcase<br>- Boundary testcase               |
| 2   | Use Case / User Flow       | Ví dụ:<br>- Đăng ký<br>- Đăng nhập<br>- Tìm kiếm sản phẩm<br>- Thêm giỏ hàng<br>- Checkout<br>- Thanh toán<br>- Theo dõi đơn hàng                                                                                                                                                                                                  | Giúp tester hiểu luồng sử dụng hệ thống                                                                |
| 3   | Mô tả giao diện            | Nội dung:<br>- Có màn hình nào<br>- Có field nào<br>- Có button nào<br>- Error message ở đâu<br><br>Ví dụ:<br>Form login có email, password, remember me, forgot password                                                                                                                                                          | Tạo testcase UI/UX:<br>- Field bắt buộc<br>- Placeholder<br>- Validate<br>- Responsive<br>- Điều hướng |
| 4   | Business Rules             | Cực kỳ quan trọng với e-commerce.<br><br>Ví dụ:<br>- Không được mua quá tồn kho<br>- Freeship khi > 300k<br>- Voucher chỉ dùng 1 lần/user<br>- COD chỉ áp dụng nội thành<br>- Guest không xem lịch sử đơn hàng                                                                                                                     | Nguồn sinh ra:<br>- Testcase logic nghiệp vụ<br>- Edge case<br>- Exception case                        |
| 5   | Database Schema            | Ví dụ:<br>- orders.status<br>- products.stock<br>- users.role                                                                                                                                                                                                                                                                      | - Verify dữ liệu lưu đúng<br>- Check trạng thái order<br>- Check soft delete<br>- Check inventory      |
| 6   | API Document               | Nội dung:<br>- Swagger<br>- Postman Collection<br>- API Spec                                                                                                                                                                                                                                                                       | - Test API độc lập<br>- Test response<br>- Test auth/token<br>- Test status code                       |
| 7   | Phân quyền hệ thống        | Cơ cấu:<br>- Guest<br>- Customer<br>- Staff<br>- Admin<br>- Super Admin                                                                                                                                                                                                                                                            | Kiểm tra mỗi role được:<br>- Xem gì<br>- Sửa gì<br>- Xóa gì                                            |
| 8   | Danh sách chức năng        | Ví dụ:<br>- Auth: Login/Register<br>- Product: CRUD sản phẩm<br>- Cart: Thêm/xóa giỏ hàng<br>- Order: Checkout/Payment<br>- Admin: Quản lý user                                                                                                                                                                                    | Mapping testcase theo module                                                                           |
| 9   | Acceptance Criteria        | Giống “điều kiện pass”.<br><br>Ví dụ:<br>Khi checkout thành công thì:<br>- Tạo order<br>- Trừ tồn kho<br>- Gửi email<br>- Chuyển trạng thái pending                                                                                                                                                                                | Verify theo checklist này                                                                              |
| 10  | Environment / Test Account | Cần có:<br>- URL test<br>- Tài khoản admin<br>- Tài khoản user<br>- Dữ liệu mẫu<br><br>Ví dụ:<br>- [admin@gmail.com](mailto:admin@gmail.com)<br>- [user01@gmail.com](mailto:user01@gmail.com)                                                                                                                                      | - URL test<br>- Tài khoản admin<br>- Tài khoản user<br>- Dữ liệu mẫu                                   |
